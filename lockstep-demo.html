A system-of-record for your team's coding agents

Keep two developers' AI agents in lockstep — without sharing a line of source code.

When your teammate's Claude (or Codex, or Gemini) changes an API, your agent finds out, knows what changed and why, and stops building on stale assumptions — automatically.

The problem

Code syncs. Understanding doesn't.

Two devs, two coding agents, one codebase. Git shares the files — but each agent learns the code in isolation. Decisions, intent, and the contract between services live in one person's head and never reach the other's agent.

🧑‍💻 Naman · order-service · Claude Code

⟷

🧑‍💻 Friend · product-service · Codex

The fix isn't more chat. It's a shared, verified record both agents read and write — automatically.

Why it's different

Built for collaborators, not one power-user.

🔒

Source never leaves

Only metadata crosses the wire — summaries, contracts, decisions. Your actual code stays on your machine. The reason agency↔client work is even possible.

🔀

Cross-vendor

Claude Code, Codex, Gemini CLI — all speak to the same record via hooks + MCP. The one thing a single model vendor can't serve.

✓

Verified, not asserted

The shared contract is checked against your real local types before it's published — so teammates build on truth, not on an agent's guess.

🧭

Ownership routing

CODEOWNERS decides who owns what. Changes route to consumers; questions route to owners. No @-mention guesswork.

🛡️

Enforced at the PR

A CI check reconciles your diff against the ledger. Break a binding decision and the merge is blocked — soft coordination becomes a hard gate.

🧠

Ask the record

“Why do we use idempotency keys?” Your agent queries the ledger first and answers instantly — only escalating to a human when it's genuinely new.

Clear your doubts

Ask the product anything.

Type a question the way you'd validate it — “does my code leave my machine?”, “does it work with Cursor?”, “what if two of us edit the same API?”. Answers come straight from the design.
